# 流式输出与 Token 统计优化

## 概述

为个人助手机器人添加流式输出和 Token 使用统计功能，提升用户体验并便于成本控制。

## 完成时间

2026-03-06

## 实施内容

### 1. 流式输出

#### 新增函数

**`bot/assistant/llm.py`**

- `call_claude_api_stream()`: 流式调用 Claude API
  - 使用 `client.messages.stream()` 进行流式调用
  - 逐步返回文本增量、工具调用和 token 使用信息
  - 支持 tool use（工具调用）

- `call_claude_with_memory_tools_stream()`: 支持记忆工具的流式调用
  - 第一轮使用流式输出，提升响应速度
  - 后续轮次（tool use 场景）使用非流式
  - 累计所有轮次的 token 使用

#### 更新函数

**`bot/assistant/llm.py`**

- `call_claude_api()`: 添加 token 使用统计
  - 返回值新增 `usage` 字段：`{"input_tokens": 100, "output_tokens": 50}`

- `call_claude_with_memory_tools()`: 保持向后兼容
  - 非流式版本，用于测试和向后兼容
  - 返回纯文本字符串

### 2. Token 使用统计

#### 显示格式

在每次回复末尾添加 token 使用信息：

```
💰 Token 使用: 150 输入 + 80 输出 = 230 总计
```

#### 统计范围

- 包含所有 API 调用的 token 使用（包括 tool use 循环）
- 输入 token：用户消息 + 系统提示 + 对话历史 + 记忆注入
- 输出 token：AI 回复 + 工具调用

### 3. 用户体验优化

#### 流式更新策略

- 每收到约 100 字符更新一次消息（避免过于频繁）
- 最终更新包含完整文本 + token 统计
- 如果消息过长导致编辑失败，自动删除旧消息并发送新消息

#### 状态提示

- 发送消息时显示 "🤔 正在思考..."
- 流式输出时逐步更新消息内容
- 完成后显示完整回复 + token 统计

## 技术实现

### 流式输出事件格式

```python
# 文本增量
{"type": "text", "text": "..."}

# 工具调用开始
{"type": "tool_use_start", "id": "...", "name": "..."}

# 工具调用完整信息
{"type": "tool_use", "id": "...", "name": "...", "input": {...}}

# Token 使用统计
{"type": "usage", "input_tokens": 100, "output_tokens": 50}
```

### 处理流程

```
用户发送消息
    ↓
显示"正在思考..."
    ↓
流式调用 Claude API
    ↓
逐步更新消息内容（每 ~100 字符）
    ↓
收到完整回复 + token 统计
    ↓
最终更新：完整文本 + token 信息
    ↓
保存到对话历史
```

## 性能考虑

### 流式输出优势

1. **更快的首字响应**：用户无需等待完整回复生成
2. **更好的交互体验**：实时看到 AI 的思考过程
3. **减少等待焦虑**：流式输出让用户知道系统正在工作

### 更新频率控制

- 每 ~100 字符更新一次（通过 `len(response_text) % 100 < len(event["text"])` 判断）
- 避免过于频繁的 API 调用（Telegram 有速率限制）
- 平衡响应速度和 API 调用次数

### Token 统计开销

- Token 统计由 API 直接返回，无额外计算开销
- 累计多轮 token 使用，准确反映实际成本
