# 1.1.13

本版重点补齐 assistant proposal patch 工作流、安装更新细节和若干运行态问题。

## Assistant

- assistant proposal patch 支持聊天式任务流转，可直接从 proposal 发起 patch 生成任务。
- patch 生成改为 SSE 流式返回，proposal 详情页新增 transcript，可查看 CLI 的 status、trace、done 和 error。
- assistant proposal 页补充 dirty 工作区提示、生成日志、失败原因和 dry-run 持久化结果，刷新后状态恢复更稳定。
- patch 生成支持取消、清理后重生成，并补了 stale worktree 恢复、超时和取消后的清理逻辑。
- bot 列表修正 assistant 状态判断：有排队任务时显示“处理中”，不再误显示“运行中”。

## 设置和终端

- 设置页 Git 代理支持直接输入 `host:port`。
- 若只填写端口，默认解析为 `127.0.0.1:port`。
- 终端预设图标做了增强，便于区分常用动作。

## 安装和更新

- 安装脚本增加插件安装提示，首次安装路径更完整。
- 自动应用已下载更新前会先检查当前版本；若当前已是目标版本，则跳过重复应用。
- `.web_tunnel_state.json` 不再进入 Git 跟踪，减少本地运行噪音。

## 稳定性

- 修复主 Web 启动链路若干问题，补强对应测试。
- proposal patch、安装脚本、updater、Web API 和前端 real client 补了更多回归测试。
