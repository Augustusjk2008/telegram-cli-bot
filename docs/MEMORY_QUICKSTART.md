# 长期记忆系统快速开始

## 启用记忆功能

记忆功能在 Phase 1 完成后自动启用，无需额外配置。

## 基本使用

### 1. 自动记忆

助手会在对话中自动记住重要信息：

```
你: 我叫张三，是一名软件工程师
助手: 你好张三！很高兴认识你。

[助手自动记住：用户名叫张三，是一名软件工程师]
```

### 2. 查看记忆

使用 `/memory` 命令查看所有记忆：

```
你: /memory
助手:
📝 你的记忆列表：

👤 个人信息
  • 用户名叫张三，是一名软件工程师
    #姓名 #职业
    ID: user_123_1234567890
```

### 3. 搜索记忆

使用 `/memory_search` 搜索特定内容：

```
你: /memory_search 工作
助手:
🔍 搜索结果（关键词: 工作）：

• 用户在北京工作
  #地点 #工作
  ID: user_123_1234567892
```

### 4. 删除记忆

使用 `/memory_delete` 删除指定记忆：

```
你: /memory_delete user_123_1234567890
助手: ✅ 已删除记忆: user_123_1234567890
```

### 5. 清空记忆

使用 `/memory_clear` 清空所有记忆：

```
你: /memory_clear
助手: ✅ 已清空 3 条记忆
```

## 手动编辑记忆

记忆文件位于 `bot/data/memories.json`，可以直接编辑：

```json
{
  "version": "1.0",
  "memories": [
    {
      "id": "user_123_1709712000",
      "user_id": 123,
      "content": "用户名叫张三，是一名软件工程师",
      "category": "personal",
      "created_at": "2026-03-06T10:00:00",
      "updated_at": "2026-03-06T10:00:00",
      "tags": ["姓名", "职业"]
    }
  ]
}
```

### 字段说明

- `id`: 唯一标识（不要修改）
- `user_id`: Telegram 用户 ID（不要修改）
- `content`: 记忆内容（可以修改）
- `category`: 分类（personal/preference/work/fact/other）
- `created_at`: 创建时间（可以不修改）
- `updated_at`: 更新时间（修改后建议更新）
- `tags`: 标签列表（可以添加/删除）

### 修改示例

1. 修改记忆内容：
```json
"content": "用户名叫张三（小张），是一名高级软件工程师"
```

2. 添加标签：
```json
"tags": ["姓名", "职业", "级别"]
```

3. 修改分类：
```json
"category": "work"
```

修改后保存文件，立即生效，无需重启。

## 记忆分类

- `personal`: 个人信息（姓名、年龄、背景等）
- `preference`: 偏好设置（喜好、习惯等）
- `work`: 工作相关（职业、项目、技能等）
- `fact`: 重要事实（地点、时间、事件等）
- `other`: 其他

## 注意事项

1. **隐私保护**：记忆存储在本地，不会上传到云端
2. **用户隔离**：每个用户的记忆完全独立
3. **自动提取**：记忆提取在后台进行，不影响对话速度
4. **精炼原则**：助手只记住重要信息，避免记忆过多无用内容

## 故障排除

### 记忆没有自动保存

1. 检查 `ANTHROPIC_API_KEY` 是否正确设置
2. 查看日志中是否有错误信息
3. 确认对话内容包含值得记住的信息

### 记忆文件损坏

如果 `memories.json` 文件损坏，可以手动修复或删除：

```bash
# 备份
cp bot/data/memories.json bot/data/memories.json.bak

# 删除（会自动重新创建）
rm bot/data/memories.json
```

### 记忆过多

使用 `/memory_clear` 清空所有记忆，或手动编辑 `memories.json` 删除不需要的记忆。
