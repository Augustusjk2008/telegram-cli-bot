# Phase 2: 长期记忆系统

## 概述

Phase 2 为个人助手机器人添加了长期记忆功能，使助手能够跨会话记住用户信息，提供个性化服务。

## 核心特性

### 1. 自动记忆提取

助手会在每次对话后自动分析对话内容，提取值得长期记住的信息：

- 个人信息（姓名、职业、背景等）
- 用户偏好（喜好、习惯等）
- 工作相关信息
- 重要事实

提取过程在后台异步进行，不影响对话响应速度。

### 2. 记忆注入

每次对话时，助手会自动加载用户的长期记忆，注入到对话上下文中，使助手能够：

- 记住用户的基本信息
- 理解用户的偏好和习惯
- 提供个性化的回答

### 3. 记忆管理命令

提供完整的记忆管理命令，用户可以查看、搜索、删除记忆：

| 命令 | 说明 | 示例 |
|------|------|------|
| `/memory` | 查看所有记忆 | `/memory` |
| `/memory_search <关键词>` | 搜索记忆 | `/memory_search 工作` |
| `/memory_delete <ID>` | 删除指定记忆 | `/memory_delete user_123_1234567890` |
| `/memory_clear` | 清空所有记忆 | `/memory_clear` |

## 数据结构设计

### 记忆数据格式

记忆数据存储在 `bot/data/memories.json`，采用易于人工阅读和编辑的 JSON 格式：

```json
{
  "version": "1.0",
  "memories": [
    {
      "id": "user_123_1234567890",
      "user_id": 123,
      "content": "用户名叫张三，是一名软件工程师",
      "category": "personal",
      "created_at": "2026-03-06T10:00:00",
      "updated_at": "2026-03-06T10:00:00",
      "tags": ["姓名", "职业"]
    }
  ]
}
```

### 字段说明

- `id`: 唯一标识，格式为 `user_{user_id}_{timestamp}`
- `user_id`: Telegram 用户 ID
- `content`: 精炼的记忆内容（一句话概括）
- `category`: 分类
  - `personal`: 个人信息
  - `preference`: 偏好设置
  - `work`: 工作相关
  - `fact`: 重要事实
  - `other`: 其他
- `created_at`: 创建时间（ISO 格式）
- `updated_at`: 最后更新时间（ISO 格式）
- `tags`: 关键词标签列表，用于检索

### 设计原则

1. **精炼**：每条记忆只保留核心信息，避免冗余
2. **准确**：通过 AI 提炼，确保信息准确性
3. **易于人工修改**：使用 JSON 格式，结构清晰，支持手动编辑

## 技术实现

### 核心模块

#### 1. `bot/assistant/memory.py`

记忆存储和管理模块：

- `Memory`: 记忆数据类
- `MemoryStore`: 记忆存储管理器
  - `add_memory()`: 添加记忆
  - `get_user_memories()`: 获取用户所有记忆
  - `search_memories()`: 搜索记忆（支持关键词、分类筛选）
  - `delete_memory()`: 删除记忆
  - `clear_user_memories()`: 清空用户记忆
  - `get_recent_memories()`: 获取最近记忆（用于注入对话）

#### 2. `bot/assistant/llm.py`

新增记忆提取功能：

- `extract_memories_from_conversation()`: 从对话中提取记忆
  - 使用 Claude API 分析对话内容
  - 判断是否包含值得记住的信息
  - 返回精炼的记忆内容、分类和标签

#### 3. `bot/handlers/assistant.py`

更新助手消息处理器：

- `_build_system_prompt_with_memory()`: 构建包含记忆的系统提示词
- `_extract_and_save_memory()`: 异步提取并保存记忆
- 新增记忆管理命令处理器：
  - `cmd_memory()`: 查看记忆
  - `cmd_memory_search()`: 搜索记忆
  - `cmd_memory_delete()`: 删除记忆
  - `cmd_memory_clear()`: 清空记忆

### 工作流程

```
用户发送消息
    ↓
加载用户的长期记忆（最近 10 条）
    ↓
注入到系统提示词
    ↓
调用 Claude API 生成回复
    ↓
返回回复给用户
    ↓
异步提取记忆（后台任务）
    ↓
保存到 memories.json
```

## 使用示例

### 示例 1：自动记忆提取

```
用户: 我叫张三，是一名软件工程师，在北京工作
助手: 你好张三！很高兴认识你。作为软件工程师，你在北京工作一定很忙碌吧。

[后台自动提取并保存记忆]
- 用户名叫张三，是一名软件工程师
- 用户在北京工作
```

### 示例 2：跨会话记忆

```
[第一次对话]
用户: 我喜欢喝咖啡
助手: 好的，我记住了你喜欢喝咖啡。

[第二次对话 - 几天后]
用户: 推荐一个下午茶的地方
助手: 既然你喜欢喝咖啡，我推荐...
```

### 示例 3：查看和管理记忆

```
用户: /memory
助手:
📝 你的记忆列表：

👤 个人信息
  • 用户名叫张三，是一名软件工程师
    #姓名 #职业
    ID: user_123_1234567890

⭐ 偏好设置
  • 喜欢喝咖啡
    #饮品
    ID: user_123_1234567891

💡 使用 /memory_delete <ID> 删除指定记忆
💡 使用 /memory_search <关键词> 搜索记忆
💡 使用 /memory_clear 清空所有记忆
```

### 示例 4：搜索记忆

```
用户: /memory_search 工作
助手:
🔍 搜索结果（关键词: 工作）：

• 用户在北京工作
  #地点 #工作
  ID: user_123_1234567892
```

### 示例 5：手动编辑记忆

用户可以直接编辑 `bot/data/memories.json` 文件：

```json
{
  "version": "1.0",
  "memories": [
    {
      "id": "user_123_1234567890",
      "user_id": 123,
      "content": "用户名叫张三（修改后的内容）",
      "category": "personal",
      "created_at": "2026-03-06T10:00:00",
      "updated_at": "2026-03-06T11:00:00",
      "tags": ["姓名"]
    }
  ]
}
```

修改后立即生效，无需重启。

## 性能考虑

1. **异步提取**：记忆提取在后台异步进行，不阻塞用户对话
2. **限制注入数量**：每次对话只注入最近 10 条记忆，避免上下文过长
3. **轻量级存储**：使用 JSON 文件存储，无需额外数据库依赖

## 隐私和安全

1. **用户隔离**：每个用户的记忆完全隔离，互不干扰
2. **本地存储**：记忆数据存储在本地，不上传到云端
3. **用户控制**：用户可以随时查看、删除、清空自己的记忆
